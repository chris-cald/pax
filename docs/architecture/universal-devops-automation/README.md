# Universal DevOps Automation: Starter Document Set

This folder contains a minimal, practical document set to:

- evaluate market opportunity
- run comparative solution analysis
- seed a new project with clear product boundaries

## Who This Is For

- founder/operator validating a product thesis
- platform engineer defining a reusable internal capability
- product/engineering team preparing a v1 scope

## Document Map

- [Product Requirements](PRD.md)
- [Operating Model and Phase Contracts](OPERATING_MODEL.md)
- [Market Analysis Template](../../reference/universal-devops-automation/MARKET_ANALYSIS_TEMPLATE.md)
- [Comparative Analysis Framework](../../reference/universal-devops-automation/COMPARATIVE_ANALYSIS_FRAMEWORK.md)

## Core Workflow Phases

1. discover
2. validate
3. remediate
4. authorize
5. handoff
6. monitor

## Suggested Usage

1. Fill out the PRD to lock problem, target user, and MVP boundaries.
2. Complete the market analysis template to pressure-test demand and segment fit.
3. Run comparative analysis against build, buy, and hybrid alternatives.
4. Finalize phase contracts in OPERATING_MODEL.md before implementation.

## Minimal Success Criteria

- one target repository can reach publish-ready handoff deterministically
- human-gated steps are explicit and auditable
- phase outputs are machine-checkable and stable across reruns
